ЗАДАЧА 2: GROQ AI ИНТЕГРАЦИЯ (нативный Java-сервис)

[2.1] Новый файл: com/exteragram/messenger/groq/GroqConfig.java

[2.2] Новый файл: com/exteragram/messenger/groq/GroqManager.java
  - Синглтон-менеджер, изолированный от TGNet
  - Использует OkHttp3 (уже в проекте: 4.10.0)
  - Методы:
    * init(Context) — загрузка настроек из SharedPreferences
    * isEnabled() — активен ли (есть ключ + включён)
    * setApiKey(String) — сохранение ключа
    * setModel(String) — сохранение модели
    * setCustomPrompt(String) — кастомный промпт
    * restructureText(String text, GroqCallback callback) — основная функция
    * testConnection(GroqCallback callback) — тест ключа
    * cancelRequest() — отмена текущего запроса
  - Отдельный OkHttpClient с таймаутами 30s
  - Работает в background thread, callback на main thread
  - Не затрагивает ConnectionsManager / TGNet

[2.3] Интерфейс: com/exteragram/messenger/groq/GroqCallback.java
  - onSuccess(String restructuredText)
  - onError(String errorMessage)
  - onProcessing() — для показа прогресса

[2.4] Настройки в ExteraConfig.java

ЗАДАЧА 4: КАСТОМНЫЕ ИКОНКИ ПРИЛОЖЕНИЯ
================================================================

КОНЦЕПЦИЯ:
  - Android не позволяет менять иконку лаунчера на произвольный bitmap в рантайме
  - Решение: 3 pre-declared activity-alias ("CustomIcon1/2/3") в манифесте
    с placeholder-иконкой
  - Пользователь загружает фото → кадрирует → сохраняется в internal storage
  - В UI выбора иконок кастомные иконки отображаются (реальное фото!)
  - При выборе кастомной иконки — активируется соответствующий alias
  - На рабочем столе Android показывает статический placeholder (ограничение ОС),
    но ВНУТРИ приложения во всех UI-местах показывается реальное фото
  - Пользователь всегда знает какая иконка активна

[4.1] Добавить в AndroidManifest.xml:
  - <activity-alias android:name="com.exteragram.messenger.CustomIcon1" ...>
  - <activity-alias android:name="com.exteragram.messenger.CustomIcon2" ...>
  - <activity-alias android:name="com.exteragram.messenger.CustomIcon3" ...>
  - Все с android:icon="@mipmap/ic_launcher_custom_placeholder"

[4.2] Новый файл: com/exteragram/messenger/icons/CustomIconManager.java
  - Директория хранения: getFilesDir()/custom_icons/
  - Методы:
    * saveCustomIcon(int slot, Bitmap bitmap) → boolean
    * loadCustomIcon(int slot) → Bitmap
    * deleteCustomIcon(int slot)
    * hasCustomIcon(int slot) → boolean
    * getCustomIconCount() → int
    * getCustomIconBitmap(int slot) → Bitmap

[4.3] Новый файл: com/exteragram/messenger/icons/CustomIconCropActivity.java
  - Экран кадрирования изображения
  - Загрузка из галереи (Intent.ACTION_PICK)
  - Ручное кадрирование: зажимаемый квадратный селектор поверх изображения
  - Кнопки: "Готово" / "Отмена"
  - Результат — квадратный Bitmap 192x192 (стандарт Android mipmap-xxxhdpi)
  - Сохранение через CustomIconManager

[4.4] Расширить LauncherIconController.java
