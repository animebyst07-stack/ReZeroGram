ЗАДАЧА 1: Сообщения не отображаются ("Сообщение не поддерживается
          Вашей версией ZeroGram. Пожалуйста, обновите приложение")
-----------------------------------------------------------------

СИМПТОМ:
  В каналах (и не только) вместо содержимого сообщений — текст
  "Сообщение не поддерживается Вашей версией ZeroGram".
  В ExtraGram (ласт версия) те же сообщения отображаются нормально.

ПРИЧИНА (диагноз):
  Telegram-сервер использует TL (Type Language) схему для передачи
  объектов. При получении неизвестного типа объекта форк выводит
  заглушку "не поддерживается". Наш форк базируется на коде ExtraGram,
  но, судя по скриншотам, не поддерживает новые типы сообщений:
    - TL_message_service с новыми action-типами (Stars-подарки, Gifts)
    - Новые типы медиа (TL_messageMediaPaidMedia или аналог)
    - Sponsored messages / новые форматы закреплённых сообщений
  LAYER в нашем коде: 158 (TLRPC.java, строка 74)
  ExtraGram, вероятно, уже обновился на LAYER 166+ где эти типы есть.

ЧТО НУЖНО СДЕЛАТЬ:
  A) Найти актуальный TL-слой из официального Telegram Android:
       https://github.com/DrKLO/Telegram/blob/master/TMessagesProj/
       src/main/java/org/telegram/tgnet/TLRPC.java
     Или взять из ExtraGram последней версии (GitHub exteragram-dev).

  B) Обновить TLRPC.java — добавить отсутствующие классы для:
       - TL_messageActionGiftStars (Stars-подарки)
       - TL_messageActionPrizeStars
       - TL_messageMediaGiveaway / TL_messageMediaGiveawayResults
       - TL_messageActionPaymentSentMe (новые виды оплаты)
       - Прочие новые TL-типы из свежего слоя

  C) Обновить LAYER = 158 → актуальное значение (например, 166)

  D) Проверить MessagesController.java и MessageObject.java на
     обработку новых типов (иначе объекты будут создаваться, но
     не отображаться корректно)

ФАЙЛЫ ДЛЯ ИЗМЕНЕНИЯ:
  - TMessagesProj/src/main/java/org/telegram/tgnet/TLRPC.java
  - TMessagesProj/src/main/java/org/telegram/messenger/MessageObject.java
  - TMessagesProj/src/main/java/org/telegram/ui/Cells/ChatMessageCell.java

ПРИОРИТЕТ: КРИТИЧЕСКИЙ — пользователи не могут читать новые сообщения

ЗАДАЧА 2: Кнопка Groq AI не появляется в поле ввода чата
-----------------------------------------------------------------

СИМПТОМ:
  Groq настроен и включён (ключ вставлен, галочка включена), но
  при вводе текста в поле сообщения кнопка Groq не появляется
  ни в поле ввода, ни где-либо ещё.

ПРИЧИНА (диагноз):
  В GroqManager.java конструктор НЕ загружает настройки из SharedPrefs:

    private GroqManager() {
        httpClient = new OkHttpClient.Builder()...
        mainHandler = new Handler(Looper.getMainLooper());
        // ← prefs НЕ загружаются здесь!
    }

  Настройки загружаются только при вызове init(context):
    public void init(Context context) {
        prefs = context.getApplicationContext()...
        enabled = prefs.getBoolean(KEY_ENABLED, false);  ← дефолт false
        apiKey  = prefs.getString(KEY_API_KEY, "");
    }

  ЕСЛИ init() не вызывается при старте приложения до создания
  ChatActivityEnterView — isEnabled() вернёт false, и кнопка
  останется GONE навсегда, даже при введённом тексте.

  Дополнительная проблема: updateGroqInputButton() вызывается
  ТОЛЬКО в onTextChanged (строка 4301). Если после вызова init()
  поле уже содержит текст — кнопка не появится, пока текст не
  изменится.

ЧТО НУЖНО СДЕЛАТЬ:
  A) Убедиться, что GroqManager.getInstance().init(context) вызывается
     при старте приложения. Найти ApplicationLoader.java или
     LaunchActivity.java и добавить вызов:
       GroqManager.getInstance().init(context);
     В строке Application.onCreate() или LaunchActivity.onCreate()

  B) После добавления кнопки в attachLayout вызвать
     updateGroqInputButton() сразу (не только в onTextChanged):
       // После строки ~2061: attachLayout.addView(groqInputButton...)
       groqInputButton.post(() ->
           updateGroqInputButton(!TextUtils.isEmpty(
               messageEditText != null ? messageEditText.getText() : null)));

  C) Также проверить, что в GroqPreferencesActivity при включении
     тумблера вызывается notifyListeners() или broadcast, чтобы
     ChatActivityEnterView перерисовал кнопку немедленно

ФАЙЛЫ ДЛЯ ИЗМЕНЕНИЯ:
  - TMessagesProj/src/main/java/org/telegram/messenger/ApplicationLoader.java
    (найти Application.onCreate() — добавить init Groq)
  - TMessagesProj/src/main/java/org/telegram/ui/Components/ChatActivityEnterView.java
    (строка ~2061 — добавить post-вызов updateGroqInputButton)
  - TMessagesProj/src/main/java/com/ZeroGram/ReZeroGram/groq/GroqPreferencesActivity.java
    (при изменении enabled — уведомить ChatActivityEnterView)

ЗАДАЧА 3А: Переименовать "exteraGram" → "ZeroGram+" в настройках
-----------------------------------------------------------------

СИМПТОМ:
  В разделе настроек отображается "exteraGram Preferences"
  вместо "ZeroGram+ Preferences". Также в других строках
  встречается старое имя "exteraGram".

ПРИЧИНА:
  В файле extera.xml строки не переименованы:
    <string name="Preferences">exteraGram Preferences</string>
    <string name="PauseOnMinimizeInfo">...exteraGram is minimized.</string>
    <string name="ResetSettingsBulletin">**exteraGram** Preferences...</string>

ЧТО НУЖНО СДЕЛАТЬ:
  В файле TMessagesProj/src/main/res/values/extera.xml заменить:

    ДО:   <string name="Preferences">exteraGram Preferences</string>
    ПОСЛЕ: <string name="Preferences">ZeroGram+ Preferences</string>

    ДО:   ...exteraGram is minimized.</string>
    ПОСЛЕ: ...ZeroGram+ is minimized.</string>

    ДО:   **exteraGram** Preferences have been reset.
    ПОСЛЕ: **ZeroGram+** Preferences have been reset.

  Проверить все другие локализации (values-ru/, values-be/, etc.) —
  там тоже могут быть строки с "exteraGram".

ФАЙЛЫ ДЛЯ ИЗМЕНЕНИЯ:
  - TMessagesProj/src/main/res/values/extera.xml  (главный)
  - TMessagesProj/src/main/res/values-*/extera.xml (локализации)

ЗАДАЧА 3Б: Добавить доступ к настройкам иконки приложения
           рядом с Groq AI в главном меню настроек
-----------------------------------------------------------------

СИМПТОМ:
  Класс IconSelectorAlert.java существует в проекте
  (TMessagesProj/src/main/java/com/ZeroGram/ReZeroGram/components/),
  но пункт смены иконки приложения нигде не доступен из интерфейса.
  Пользователь хочет видеть эту опцию в главном меню настроек
  рядом с пунктом "Groq AI".

ЧТО НУЖНО СДЕЛАТЬ:
  A) В MainPreferencesActivity.java добавить новую строку в список:
       private int appIconRow;  // объявить рядом с groqRow

     В createRows() добавить:
       appIconRow = newRow();

     В onItemClick() обработать нажатие:
       } else if (position == appIconRow) {
           new IconSelectorAlert(getParentActivity(), ...);
       }

     В onBindViewHolder() отобразить ячейку:
       } else if (position == appIconRow) {
           textCell.setTextAndIcon(
               LocaleController.getString("AppIcon", R.string.AppIcon),
               R.drawable.msg_theme, false);
       }

  B) Добавить строку в extera.xml:
       <string name="AppIcon">Иконка приложения</string>

  C) Убедиться, что AndroidManifest.xml содержит <activity-alias>
     для каждой иконки-варианта (иначе смена иконки не заработает)

ФАЙЛЫ ДЛЯ ИЗМЕНЕНИЯ:
  - TMessagesProj/src/main/java/com/ZeroGram/ReZeroGram/preferences/
      MainPreferencesActivity.java
  - TMessagesProj/src/main/res/values/extera.xml
  - TMessagesProj/src/main/AndroidManifest.xml  (проверить alias-ы)
